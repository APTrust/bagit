SetupQuestion

SetupQuestion

SetupQuestion is a question to be asked by a setup module. DART presents these questions one at a time, and the user must provide a valid answer before proceeding to the next question.

You'll typically want to copy the responses from these questions into some persistent DART setting.

Constructor

new SetupQuestion(opts, id)

Parameters:
Name Type Attributes Description
opts object

An object containing setup options.
id string

A unique identifier for this question. This will become the ID of the questions HTML form input and the SetupBase#afterEachQuestion will use it to identify which question was just answered. You should prepend these with "q_" to prevent HTML ID conflicts.
opts.question string

The question to ask the user.
opts.mapsToProperty object

An object that describes where the initial value for this question should come from and where it should be saved. See SetupQuestion#mapsToProperty, which includes a full description and examples.
opts.errMessage string

An error message to display if the user's response is invalid.
opts.heading string <optional>

An optional heading to display above the question.
opts.options Array.<string> <optional>

An optional list of allowed values. If specified, the user will see an HTML select list instead of a text input. The select list will contain these values and an empty option.
opts.dataType string <optional>

The type to which the user's response should be cast. Valid values are 'number' and 'boolean'. If the dataType is string you don't need to include this param, as values are strings by default.
opts.validation string <optional>

An object describing how to validate the user's response to this question. See SetupQuestion#validation for more info.

mapsToProperty :object

An object that defines what setting value or BagItProfile tag value the question maps to. When the question appears, it will take its initial value from the specified setting/property, or from the specified BagItProfile tag value. When the user provides a valid response, the value of that response will be assigned to the setting/property or tag value.

validation :object

An object that describes how this question should be validated. The object has the following properties.

required: Set this to true if the user must respond. Default is false.

pattern: A regular expression pattern against which to validate the user's response. You can set this to a quoted pattern like "/\w+/" or use one of the values defined in Constants, such as:

  • Constants.RE_DOMAIN
  • Constants.RE_EMAIL
  • Constants.RE_FILE_PATH_POSIX
  • Constants.RE_FILE_PATH_WINDOWS
  • Constants.RE_FILE_PATH_ANY_OS
  • Constants.RE_IPV4

min: If the response should be numeric, this specifies that it must be greater than or equal to this value.

max: If the response should be numeric, this specifies that it must be less than or equal to this value.

Source:
  • forms/setup_question.js, line 141
Example
// Create a validator that requires an answer and will accept
// any answer (no min, max, or pattern).
opts.validator = {
    required: true
}

// Validator that does not require a response, but if given, the
// response must be an email address. Note that RE_EMAIL comes
// from the Constants module.
opts.validator = {
    pattern: 'RE_EMAIL'
}

// Validator that requires a string of 4 or more hex digits.
// The string pattern will be converted to a RegExp.
opts.validator = {
    required: true,
    pattern: '[a-fA-F0-9]{4,}'
}

// Validator requiring a number between 1 and 100 inclusive.
opts.validator = {
    required: true,
    min: 1,
    max: 100
}

_setValidationRegExp()

This sets the RegExp object for validation if the validation object includes a pattern property. If validation.pattern is set to any of the following strings, this assigns the RegExp from the Constants module: 'RE_DOMAIN', 'RE_IPV4', 'RE_FILE_PATH_POSIX', 'RE_FILE_PATH_WINDOWS', 'RE_FILE_PATH_ANY_OS', 'RE_EMAIL'.

If validation.pattern is set to any other string, this converts the string to a regexp.

The resulting RegExp is assigned to this.validation.regexp.
